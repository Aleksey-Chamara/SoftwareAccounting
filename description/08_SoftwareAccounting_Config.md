# Конфигурация системы учета ПО (IT Audit System)

> Настройки приложения, роли пользователей и бизнес-правила.

## Конфиг-параметры (appsettings.json)

> Настройки, управляемые через файл конфигурации.

| **Параметр**                          | **Тип (C#)**     | **Значение по умолчанию**               | **Описание**                                                              |
| ------------------------------------- | ---------------- | --------------------------------------- | ------------------------------------------------------------------------- |
| `ConnectionStrings:DefaultConnection` | `string`         | `"DataSource=app.db"`                   | Строка подключения к БД (SQLite).                                         |
| `Logging:LogLevel:Default`            | `string`         | `"Information"`                         | Уровень логирования по умолчанию.                                         |
| `Logging:LogLevel:Microsoft.AspNetCore` | `string`       | `"Warning"`                             | Уровень логирования для ASP.NET Core (предупреждения и выше).             |
| `AllowedHosts`                        | `string`         | `"*"`                                   | Разрешенные хосты для публикации (`*` — все).                            |

## Пользователи и роли (Identity)

| **Роль** | **Права**                                                                 | **Учетные данные по умолчанию**              |
| -------- | ------------------------------------------------------------------------- | --------------------------------------------- |
| `Admin`  | Полный доступ: CRUD компьютеров, управление ПО (+ Other), удаление.       | `admin` / `admin` (пароль)                    |
| `User`   | Только просмотр: список компьютеров, тех. паспорт, просмотр установленного ПО. | `user` / `user` (пароль)                      |

> **Политика паролей:** Ослаблена для тестирования (мин. длина 3 символа, без требований к регистру, цифрам и спецсимволам).

## Где храним данные

| **app.db** (SQLite)                                                   | **Миграции (EF Core)**                                                 |
| --------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| Основной файл БД. Содержит таблицы: `Computers`, `Softwares`, `InstalledSoftwares`, а также системные таблицы `AspNetUsers`, `AspNetRoles` и пр. | Управляются через `Microsoft.EntityFrameworkCore.Tools`. Хранятся в папке `Migrations`. |

## Бизнес-правила и ограничения (Constraints)

| Ограничение                    | Описание                                                                                                                                 |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **Кастомное ПО (Other)**       | Если ПО отсутствует в справочнике `Softwares`, пользователь (Admin) может добавить его вручную. `SoftwareId` остается `null`, заполняется `CustomSoftwareName`. |
| **Лицензии**                   | Поддерживаются 4 типа: `Licensed` (Лицензия), `Free` (Free), `Shareware` (Shareware), `Trial` (Trial).                                   |
| **Уникальность**               | Уникальность `MAC-адреса` или `Owner` **не** проверяется на уровне кода (нет атрибутов `[Unique]`).                                     |
| **Сортировка**                 | Доступна для списка компьютеров (по Владельцу, MAC, дате проверки) и для ПО в деталях компьютера (по имени, лицензии, типу, дате установки). |
| **Авторизация**                | Все контроллеры, кроме `AccountController`, требуют авторизации (`[Authorize]`). Действия записи требуют роли `Admin`.                  |

## Политика обработки ошибок

**Что показывает пользователю:**

*   **Ошибка входа:** `"Неверный логин или пароль"` на странице `/Account/Login`.
*   **Общая ошибка БД:** Перенаправление на `/Home/Error` (в production). В development — страница с деталями исключения.
*   **Ошибка 404 (Not Found):** Стандартная страница NotFound при попытке редактирования/удаления несуществующей записи.
*   **Ошибка валидации:** Подсветка полей и сообщения в представлениях `Create`/`Edit` (через `asp-validation-summary`).

**Что логируется:**

*   Ошибки инициализации БД (при первом запуске) через `ILogger<Program>`.
*   `DbUpdateConcurrencyException` при одновременном редактировании (выбрасывается повторно, если запись не найдена).

## Инициализация БД (DbInitializer)

При первом запуске автоматически:

1.  Создаются роли `Admin` и `User`.
2.  Создаются пользователи:
    *   `admin` (пароль `admin`) в роли `Admin`.
    *   `user` (пароль `user`) в роли `User`.
3.  Заполняется справочник ПО:
    *   `Windows 11 Pro` (Licensed, OS)
    *   `MS Office 2021` (Licensed, Office)
    *   `WinRAR` (Shareware, Archiver)
    *   `VS Code` (Free, Development)
    *   `Kaspersky Endpoint` (Trial, Antivirus)
4.  Создаются тестовые компьютеры (директор, бухгалтер, разработчик) с привязкой к ПО и кастомными программами (например, `1С: Бухгалтерия 8.3`).
