> **Более подробное описание проекта**
> Это веб-приложение на платформе ASP.NET Core для инвентаризации, учета и формирования технической отчетности по компьютерам организации и установленному на них программному обеспечению.
> 
> Система не сканирует сеть автоматически, а представляет собой структурированную базу данных (паспортизацию), позволяющую администраторам вести учет рабочих станций, отслеживать легитимность используемых лицензий (включая кастомный софт категории Other) и быстро генерировать технические паспорта.

## Про пользователя

Основная аудитория — это системные администраторы, сотрудники отделов ИТ-обеспечения и инвентаризационные комиссии:
- **Системные администраторы (Admin):** Заводят компьютеры в систему, актуализируют списки установленного ПО, управляют глобальным справочником разрешенных программ.
- **ИТ-директора и Аудиторы (User):** Имеют доступ к просмотру базы, поиску, сортировке и выгрузке технических паспортов/отчетов для проверки лицензионной чистоты.

**Зачем:**
Чтобы оперативно видеть, на каком компьютере установлена нелицензионная или триальная программа, у кого из сотрудников устарели данные проверки, и быстро формировать отчеты для руководства или проверяющих органов без ручного обхода рабочих мест.

## Предполагаемый функционал
### Ключевые функции

1. **Гибкий учет конфигураций:** Закрепление ПК за конкретным владельцем (ФИО), фиксация уникального MAC-адреса и даты последней проверки.
2. **Умный контроль лицензий:** Разделение ПО по типам лицензирования (`Licensed`, `Free`, `Shareware`, `Trial`) для контроля рисков.
3. **Поддержка стороннего софта (Other):** Возможность привязать к компьютеру программу, отсутствующую в общем справочнике, через текстовое поле `CustomSoftwareName`.
4. **Многокритериальная сортировка:** Мгновенная фильтрация и сортировка списков по владельцу, MAC-адресу, типу лицензии и дате проверки.
5. **Технический паспорт:** Генерация единой детальной карточки компьютера со всей историей установок.
6. **Разграничение прав (RBAC):** Безопасный доступ с разделением на Администраторов и Пользователей через ASP.NET Core Identity.

### Чего скорее всего не будет

1. **Автоматического сетевого сканирования (WMI/Agent):** Вся информация вносится вручную или через миграционные XML/JSON (система является каталогизатором, а не сетевым сканером).
2. **Учета комплектующих (Hardware Inventory):** Без детализации плашек оперативной памяти, моделей процессоров и серийных номеров дисков (фокус строго на связке ПК + Лицензии ПО).
3. **Интеграции с Active Directory:** На данном этапе используется локальная база пользователей Identity.
