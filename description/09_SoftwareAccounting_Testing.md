# Стратегия тестирования системы учета ПО (IT Audit System)

## 1. Уровни тестирования

### Unit-тесты (Модели, логика валидации, утилиты)

1.  **Валидация модели Computer:**
    *   Передача пустого `Owner` → `ModelState.IsValid` должен вернуть `false`.
    *   Передача некорректного `MacAddress` (например, "00-11") → ошибка валидации формата (если добавлен `[RegularExpression]`).
    *   Передача `LastCheckDate` в будущем → ошибка валидации `[Future]` (если добавлен атрибут).

2.  **Валидация модели Software:**
    *   Передача `Name` длиной 1 символ → проверка атрибута `[MinLength]`.
    *   Передача `LicenseType` со значением вне диапазона `enum` → проверка обработки.
    *   Передача пустого `SoftwareType` → проверка установки значения по умолчанию `"General"`.

3.  **Логика InstalledSoftware:**
    *   Установка `SoftwareId = null` и заполнение `CustomSoftwareName` → свойство `IsOther` должно вернуть `true`.
    *   Установка `SoftwareId = 5` и `CustomSoftwareName = null` → свойство `IsOther` должно вернуть `false`.
    *   Передача и `SoftwareId`, и `CustomSoftwareName` → тест на бизнес-правило (должен быть запрещено или приоритет у `SoftwareId`).

4.  **Сортировка Computers (логика контроллера):**
    *   Передача `sortOrder = "owner_desc"` → проверка применения `OrderByDescending(s => s.Owner)`.
    *   Передача `sortOrder = "Mac"` → проверка применения `OrderBy(s => s.MacAddress)`.
    *   Передача `sortOrder = null` или неизвестного значения → проверка применения сортировки по умолчанию (`OrderBy(s => s.Owner)`).

5.  **Сортировка ПО в деталях компьютера:**
    *   Передача `sortOrder = "name_desc"` → проверка сортировки с учетом `IsOther` (кастомные программы в `CustomSoftwareName`).
    *   Передача `sortOrder = "License"` → проверка сортировки по `Software.License`, при этом `null` (Other) должны быть в конце или начале.

### Integration-тесты (Связка слоев, БД, авторизация)

1.  **Подключение к SQLite:**
    *   Применение миграции `InitialCreate` → проверка создания всех таблиц (`Computers`, `Softwares`, `InstalledSoftwares`, `AspNetUsers`, `AspNetRoles` и связей).
    *   Выполнение `SaveChangesAsync()` → проверка корректной генерации `Id` (автоинкремент) для новых записей.

2.  **DbInitializer (инициализация БД):**
    *   Первый запуск приложения → проверка создания ролей `Admin` и `User`.
    *   Первый запуск приложения → проверка создания пользователей `admin` (пароль `admin`) и `user` (пароль `user`).
    *   Первый запуск приложения → проверка добавления 5 предустановленных программ в таблицу `Softwares`.
    *   Первый запуск приложения → проверка добавления 3 тестовых компьютеров с привязкой к ПО и кастомными программами (Other).

3.  **Связи в БД (Entity Framework):**
    *   Удаление `Computer` → проверка каскадного удаления связанных записей из `InstalledSoftwares` (настроено `OnDelete(DeleteBehavior.Cascade)`).
    *   Загрузка `Computer` с `Include(c => c.Installations).ThenInclude(i => i.Software)` → проверка корректной материализации навигационных свойств.

4.  **Авторизация (контроллеры):**
    *   Доступ неавторизованного пользователя к `/Computers` → проверка перенаправления на `/Account/Login`.
    *   Доступ пользователя с ролью `User` к `Computers/ManageSoftware` → проверка возврата `403 Forbidden` или перенаправления на страницу ошибки.
    *   Доступ пользователя с ролью `Admin` к `Computers/Create` → проверка успешной загрузки страницы (HTTP 200).

5.  **Аутентификация (AccountController):**
    *   Ввод правильных учетных данных (`admin`/`admin`) → проверка `result.Succeeded == true` и редиректа на `Computers/Index`.
    *   Ввод неправильных учетных данных → проверка `result.Succeeded == false` и установки `ViewBag.Error = "Неверный логин или пароль"`.
    *   Вызов `Logout` → проверка вызова `_signInManager.SignOutAsync()` и редиректа на `Login`.

### E2E / Acceptance-тесты (Сценарии пользователя)

1.  **Smoke-тест:**
    *   Запуск приложения → отображение страницы входа (`/Account/Login`) без ошибок в консоли браузера и сервера.
    *   Успешный вход под `admin` → отображение списка компьютеров (`Computers/Index`) с тремя тестовыми записями.

2.  **Сценарий: Просмотр технического паспорта**
    *   Вход под `user` → клик на кнопку "Детали" (📄) у любого компьютера → открытие страницы `Computers/Details/{id}`.
    *   Проверка отображения: Владелец, MAC-адрес, дата проверки, количество ПО.
    *   Проверка таблицы ПО: отображение названия, типа лицензии, категории, даты установки.
    *   Клик по заголовку "НАИМЕНОВАНИЕ ПО" → проверка сортировки списка по имени (A→Z, затем Z→A).

3.  **Сценарий: Добавление нового компьютера (Admin)**
    *   Вход под `admin` → переход на `Computers/Create`.
    *   Заполнение полей: Владелец = "Тестовый ПК", MAC-адрес = "AA-BB-CC-DD-EE-FF", Дата проверки = текущая.
    *   Отправка формы → редирект на список компьютеров → появление новой записи в таблице.
    *   Проверка БД: запись добавлена в таблицу `Computers`.

4.  **Сценарий: Установка ПО на компьютер из справочника (Admin)**
    *   Вход под `admin` → открыть детали компьютера → нажать "Управление ПО".
    *   В блоке "Из справочника" выбрать "VS Code" → выбрать дату → нажать "Добавить".
    *   Возврат на страницу управления → проверка появления "VS Code" в списке установленных компонентов.
    *   Возврат в техпаспорт → проверка отображения "VS Code" в таблице ПО.

5.  **Сценарий: Добавление кастомного ПО (Other) (Admin)**
    *   Вход под `admin` → открыть "Управление ПО" для компьютера.
    *   В блоке "Ручной ввод" ввести название "Custom Internal Tool 1.0" → выбрать дату → нажать "Добавить (Other)".
    *   Проверка появления записи в списке с бейджем "Other".
    *   Проверка техпаспорта: отображение "Custom Internal Tool 1.0" в таблице, тип лицензии = "Не определено", категория = "Прочее".

6.  **Сценарий: Удаление ПО с компьютера (Admin)**
    *   Вход под `admin` → открыть "Управление ПО" для компьютера.
    *   Нажать кнопку ❌ (удалить) напротив любой программы → проверка исчезновения записи из списка.
    *   Проверка техпаспорта: программа отсутствует в таблице.

7.  **Сценарий: Удаление компьютера (Admin)**
    *   Вход под `admin` → открыть список компьютеров → нажать кнопку "Корзина" (🗑️) напротив компьютера.
    *   Подтверждение удаления → редирект на список → проверка отсутствия удаленного компьютера.
    *   Проверка БД: запись удалена из `Computers`, все связанные записи удалены из `InstalledSoftwares`.

8.  **Сценарий: Редактирование компьютера (Admin)**
    *   Вход под `admin` → открыть детали компьютера → нажать "Параметры" (Edit).
    *   Изменить владельца с "Иванов И.И. (Директор)" на "Иванов И.И. (Генеральный директор)" → сохранить.
    *   Проверка: в деталях и списке отображается обновленное имя владельца.

9.  **Сценарий: Сортировка списка компьютеров**
    *   Вход под `admin` → список компьютеров.
    *   Клик по заголовку "Владелец" → проверка сортировки по алфавиту (A→Я).
    *   Повторный клик → проверка сортировки в обратном порядке (Я→A).
    *   Клик по "MAC адрес" → проверка сортировки по MAC.
    *   Клик по "Дата проверки" → проверка сортировки по дате (старые → новые → старые).

10. **Сценарий: Управление справочником ПО (Softwares)**
    *   Вход под `admin` → переход на `Softwares/Index` → просмотр списка из 5 программ.
    *   Нажать "Создать" → заполнить: Название = "Adobe Photoshop", Лицензия = "Licensed", Тип ПО = "Graphics" → сохранить.
    *   Проверка: новая программа появилась в списке и доступна для выбора в выпадающем списке "Управление ПО".

11. **Сценарий: Разграничение прав доступа**
    *   Вход под `user` → список компьютеров виден.
    *   Проверка ОТСУТСТВИЯ кнопок: "Добавить компьютер", "Удалить" (🗑️), "Параметры" (Edit), "Управление ПО".
    *   Попытка прямого перехода на `Computers/Create` → проверка редиректа на страницу входа или ошибки доступа.

12. **Сценарий: Обработка ошибки 404**
    *   Попытка открыть `Computers/Details/9999` (несуществующий ID) → проверка отображения страницы `NotFound` или редиректа с сообщением об ошибке.

## 2. Матрица покрытия требований (Traceability Matrix)

| **Описание MUST (из ТЗ)**                              | **Уровень теста** | **Суть проверки**                                                                                                                                                                                           |
| ----------------------------------------------------------- | ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Формирование списка компьютеров**                         | E2E               | Вход под `admin` → проверка отображения всех компьютеров из БД в таблице на странице `Computers/Index`.                                                                                                     |
| **Список установленных ПО для каждого компьютера**          | E2E               | Открытие деталей компьютера → проверка таблицы с ПО: название, лицензия, тип, дата установки.                                                                                                               |
| **Тип лицензии ПО (лицензионное/свободное/shareware/trial)** | Unit + Integration | Проверка `enum LicenseType` (4 значения). В `DbInitializer` создаются программы с каждым типом. В UI отображаются разные бейджи (зеленый/синий/желтый/красный).                                            |
| **Дата установки ПО**                                       | E2E               | В управлении ПО выбрать дату установки → проверка отображения в техпаспорте. Сортировка по дате установки в деталях.                                                                                        |
| **Тип ПО**                                                  | E2E               | В техпаспорте отображается категория ПО (OS/Office/Archiver/Development/Antivirus/Прочее). Сортировка по типу ПО.                                                                                           |
| **Справочник программных продуктов**                        | Integration       | Проверка таблицы `Softwares` в БД. `DbInitializer` добавляет 5 записей. Контроллер `SoftwaresController` позволяет просматривать, создавать (Admin) и сортировать список.                                   |
| **Обозначение "Other" для неизвестного ПО**                 | E2E               | Добавление кастомной программы через "Ручной ввод" → проверка появления бейджа "Other" в управлении и в техпаспорте (`SoftwareId = null`, заполнен `CustomSoftwareName`).                                   |
| **Технический паспорт компьютера**                          | E2E               | Страница `Computers/Details` с полной информацией: владелец, MAC, дата проверки, количество ПО, таблица ПО с возможностью сортировки.                                                                       |
| **Выдача отчета**                                           | — (не реализовано) | **Пробел:** В ТЗ указана выдача отчета, но в предоставленном коде отсутствует функциональность генерации/экспорта отчета (например, в PDF/Excel). Требуется доработка.                                      |
| **Сортировка компьютеров по владельцам**                    | Unit + E2E        | Проверка параметра `sortOrder="owner_desc"` в контроллере. В UI клик по заголовку "Владелец" меняет порядок сортировки.                                                                                     |
| **Сортировка компьютеров по MAC-адресам**                   | Unit + E2E        | Проверка параметров `sortOrder="Mac"` и `"mac_desc"`. Клик по заголовку "MAC адрес" в таблице.                                                                                                              |
| **Сортировка компьютеров по типу лицензии ПО**              | Unit + E2E        | Сортировка в `ComputersController` по полю `License` через навигационное свойство. Доступна в деталях компьютера по клику на заголовок "ТИП ЛИЦЕНЗИИ".                                                      |
| **Сортировка компьютеров по последней дате проверки**       | Unit + E2E        | Проверка параметров `sortOrder="Date"` и `"date_desc"`. Клик по заголовку "Дата проверки" в списке компьютеров.                                                                                             |
| **Хранение информации в БД**                                | Integration       | Использование SQLite + EF Core. Проверка миграций, создания таблиц, CRUD операций через `AppDbContext`. Проверка файла `app.db` после запуска.                                                              |
| **Аутентификация и авторизация**                            | Integration + E2E | Проверка: неавторизованный доступ → редирект на `/Account/Login`. Роли `Admin` (полный доступ) и `User` (только чтение). Тестовые пользователи `admin`/`admin` и `user`/`user` создаются при инициализации. |

## 3. Пробелы в тестовом покрытии (Что можно ещё сделать для нашего проекта в будущем?)

| **Проблема/Отсутствует**               | **Рекомендация**                                                                                 |
| -------------------------------------- | ------------------------------------------------------------------------------------------------ |
| **Unit-тесты для валидации моделей**   | Написать тесты с использованием `System.ComponentModel.DataAnnotations.Validator` для `Computer`, `Software`, `InstalledSoftware`. |
| **Интеграционные тесты контроллеров**  | Использовать `WebApplicationFactory<Program>` для тестирования эндпоинтов с реальной БД InMemory.  |
| **Тесты на DbUpdateConcurrencyException** | Симулировать одновременное редактирование одного компьютера двумя пользователями.               |
| **Тесты на кастомную сортировку Other** | Написать Unit-тесты для логики сортировки, где `IsOther == true` влияет на порядок.               |
| **Тесты на XSS/безопасность**          | Проверить, что `CustomSoftwareName` не выводит неэкранированный HTML/JS (CSRF-защита есть).        |
| **Функциональность "Отчет"**           | Добавить экспорт в PDF/Excel.                                                                      |
| **Логирование ошибок**                 | Добавить тесты, проверяющие, что `ILogger` вызывается при `DbUpdateConcurrencyException`.          |
| **Валидация MAC-адреса**               | В текущем коде отсутствует атрибут `[RegularExpression]` для `MacAddress`. Рекомендуется добавить и написать тесты. |
