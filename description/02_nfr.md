> Тут описаны нефункциональные требования: производительность, безопасность и специфика развертывания.

| Критерий | Расшифровка |
| :--- | :--- |
| **Производительность** | <ul><li>Рендеринг главной таблицы компьютеров (до 500 записей) должен занимать не более 200 мс.</li><li>Принудительная сортировка больших списков должна отрабатывать мгновенно за счет оптимизированных SQL-запросов (`OrderDescending`).</li><li>Использование легковесного движка SQLite обеспечивает быстрый отклик без накладных расходов на сетевой стек СУБД.</li></ul> |
| **Надёжность** | <ul><li>Использование Entity Framework Core гарантирует транзакционность: при ошибке привязки софта запись в `InstalledSoftwares` не создается, целостность связей ПК-ПО не нарушается.</li><li>Автоматический вызов `DbInitializer` при старте восстанавливает структуру таблиц и дефолтных пользователей, если БД пуста.</li></ul> |
| **Хранение данных** | <ul><li>Все данные хранятся локально в файле `app.db` в корневой директории приложения.</li><li>Файлы стилей, скриптов (Bootstrap, jQuery) хранятся локально в папке `wwwroot/lib`, обеспечивая полную автономность работы сервера без внешних CDN.</li></ul> |
| **Безопасность** | <ul><li>Все контроллеры (кроме `Account`) защищены атрибутом `[Authorize]`. Анонимный доступ запрещен.</li><li>Для удобства локального тестирования и развертывания в закрытом контуре требования к паролям Identity снижены: `RequireDigit = false`, `RequiredLength = 3`, `RequireLowercase = false` и т.д.</li></ul> |
| **Специфика среды** | <ul><li>Кроссплатформенность благодаря .NET 9.0 (возможность запуска на Windows Server под IIS, а также на Linux под Nginx в Docker-контейнере).</li></ul> |
