# Анализ нагрузки и производительности системы

> Примечание: Система учёта программного обеспечения работает в локальной сети организации и использует SQLite в качестве базы данных. Основная нагрузка приходится на операции чтения/записи БД и формирование технических паспортов компьютеров.

## Сценарии нагрузки на систему

| S1 (Массовое заполнение реестра) | S2 (Формирование технических паспортов) | S3 (Одновременная работа пользователей) |
|----------------------------------|------------------------------------------|------------------------------------------|
| Администратор импортирует данные о 1000+ компьютерах и тысячах записей ПО. | Пользователь открывает технический паспорт компьютера со 100+ программами. | Несколько сотрудников одновременно работают с реестром. |

## Оценка частоты операций

При заполнении базы для 1000 компьютеров и 20 программ на каждом создаётся до 20 000 записей.

При открытии технического паспорта выполняется загрузка компьютера, списка ПО и сортировка данных.

Большинство операций являются операциями чтения, поэтому нагрузка на SQLite остаётся невысокой.

## Меры защиты и оптимизации

- Использование Entity Framework Core.
- Оптимизированная загрузка связанных сущностей через Include().
- Индексация полей Owner, MacAddress, ComputerId и SoftwareId.
- Ограничение операций записи только для роли Admin.
- Возможность пагинации при больших объёмах данных.
- Использование транзакций SQLite.

## Отказоустойчивость

- Проверка обязательных полей.
- Контроль типов лицензий.
- Проверка корректности дат.
- Разграничение доступа по ролям.
- Автоматический откат транзакций при ошибках.

## Вывод

Система обеспечивает стабильную работу при хранении нескольких тысяч компьютеров и десятков тысяч записей установленного программного обеспечения.
