> Спецификация пользовательского интерфейса веб-приложения и сценарии работы.

## Список основных экранов

### 1.1 - Страница авторизации (`Account/Login`)
- Минималистичное окно по центру экрана.
- Поля: Имя пользователя, Пароль.
- Кнопка «Вход». Ошибки валидации выводятся над полями формы.

### 1.2 - Реестр компьютеров (`Computers/Index`)
- **Верхняя панель:** Отображение текущего пользователя, кнопка «Выйти», кнопка «Справочник ПО» и кнопка «Добавить компьютер» (только для Admin).
- **Центральная часть:** Информационная таблица со столбцами: *Владелец, MAC-адрес, Последняя проверка*. Заголовки являются кликабельными ссылками для переключения сортировки (`sortOrder`).
- **Экшены:** Напротив каждой строки доступны кнопки *Паспорт (Details)*, *Редактировать*, *Удалить*.

### 1.3 - Технический паспорт (`Computers/Details`)
- Блок общей информации о ПК (Владелец, MAC, Дата контроля).
- Таблица установленного ПО: Название, Категория, Тип лицензии, Дата установки. Кастомное ПО помечается лейблом `Other`.
- **Панель управления софтом (для Admin):** Разделена на две карточки:
  1. *Выбор из списка:* Выпадающий список (Drop-down) стандартного ПО + ввод даты.
  2. *Ручной ввод:* Текстовое поле для ввода произвольного имени программы (для ПО типа `Other`).

## Основные пользовательские сценарии

### Сценарий 1: Аудит и выявление нелицензионного ПО
1. ИТ-аудитор заходит под учетной записью `user`.
2. Переходит в карточку интересующего компьютера (нажимает *Паспорт*).
3. Просматривает таблицу программ. Видит софт со статусом лицензии `Trial` или `Shareware`.
4. Сортирует софт по типу лицензии, чтобы поднять проблемные программы вверх списка.
5. Фиксирует дату установки для последующего уведомления сотрудника.

### Сценарий 2: Добавление нового рабочего места и софта "Other"
1. Администратор (`admin`) нажимает на главной странице кнопку *Добавить компьютер*.
2. Вводит: ФИО сотрудника, MAC-адрес, ставит сегодняшнюю дату проверки. Нажимает *Сохранить*.
3. В списке нажимает кнопку *Паспорт* для только что созданного ПК.
4. В блоке «Ручной ввод» пишет *"1С: Бухгалтерия 8.3"*, указывает дату установки и нажимает *Добавить*.
5. Система обновляет страницу, программа успешно отображается в паспорте как кастомный софт.

## 5. Типовые ошибки и обработка исключений

1. **Ошибка авторизации:**
   - *Условие:* Введен неверный пароль.
   - *UI:* Страница перезагружается, выводится текст: *"Неверный логин или пароль"*.
2. **Попытка несанкционированного доступа (Нарушение прав):**
   - *Условие:* Пользователь с ролью `User` пытается вручную перейти по адресу `/Computers/Create` или `/Computers/Delete/5`.
   - *UI:* Система Identity перенаправляет пользователя на страницу отказа в доступе (`AccessDenied`) или возвращает статус 403.
3. **Отсутствие связанных данных:**
   - *Условие:* У компьютера нет ни одной установленной программы.
   - *UI:* В техническом паспорте вместо таблицы выводится аккуратное сообщение: *"На данном компьютере не зарегистрировано установленного ПО"*.
